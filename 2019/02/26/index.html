為何我選擇hexo當作個人blog?

前言

在網路上爬文學習的過程中，發現有很多人將技術文章或是教學文章都放在自己架設的部落格中。若不是放在自己的部落格中，多半是選擇 Medium 。

我也有考慮過 wordpress, Medium

最後決定使用 hexo 來架設靜態網站。

正文

這篇文章當作部落格的第一篇文章在適合不過了。

選擇 hexo 有幾個原因。

  1. 滿足自己管理網站的慾望。 (有自己的 domain name 就是開心，有種一切都是自己掌管的感覺)
  2. 相較於 wordpress 可以更加容易部署與轉移環境
  3. 以防自己的 VPS 主機有什麼三長兩短，還可以部署在 github 上
  4. markdown 語法的支援。已經習慣 markdown 了，實在不太想花心思去學其他種語法(雖然也有看到人家說 Medium 的語法很好上手，但我就是懶)

從我決定要使用 hexo 到真正部署上線花的時間約莫不到一小時，其中有比較多的時間是在看主題的設定檔，修改一些設定。

心得

基本上 hexo-cli 就能完成大部分的事情，從建立完文章模板，打開編輯器編輯文章，產生靜態檔案，部署。

對於常常接觸 cli 的人來說完全不成問題，使用第一天的心得還算滿意。

之後沒意外的話除了部署在 github.io 以外，也部署在自己的主機上。

補上一張螢幕截圖，結束第一篇文章 😄
